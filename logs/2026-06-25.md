## 5. Bugfix — verify lookup failed locally (same day)

**Symptom:** UI showed "Could not verify that ID" for order UUID and storage hash.

**Root cause:** `frontend/app/api/backend/[...path]/route.ts` forwarded to the backend
**without** `req.nextUrl.search` — so `?q=` (and `verify/recent?cursor=`) never reached FastAPI
(422 missing `q`).

**Fix:**
- Proxy: append `${req.nextUrl.search}` to backend URL.
- Backend: `_normalize_lookup_id()` — accept 64-char hex without `0x`, fix `8x` → `0x` typo.
- Frontend: map 422 to "invalid ID" error message.

**Verified:** `curl localhost:3000/api/backend/verify/lookup?q=<uuid>` and hash → HTTP 200.

**Note:** A mistyped hash (wrong chars mid-string, not just missing `0x`) still 404s — copy from
recent list or receipt link.
