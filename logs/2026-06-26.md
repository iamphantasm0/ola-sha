## Afternoon — wallet UX, PDF receipt, history tab

### 9. Onramp wallet buttons (UX fix)

**Symptom:** Onramp quote showed only “Use a different wallet” even when user had no saved wallet.

**Root causes:**
- Wallets were never persisted (banks had `save_bank`; wallets did not).
- Label always said “Use a different wallet”.
- After chat “yes” → `ONRAMP_COLLECTING_WALLET` had **no** wallet buttons.

**Fix:**
- `actions.py` — `_onramp_wallet_actions()`; label **“Add a wallet”** when empty, **“Use a different
  wallet”** when saved wallets exist; buttons in both `ONRAMP_QUOTING` and `ONRAMP_COLLECTING_WALLET`.
- `actions.py` — `save_wallet` on `ONRAMP_AWAITING_PAYMENT`; auto-save wallet on `submit_wallet`
  when logged in.
- `presenter.py` — confirm_onramp copy points to wallet buttons.
- `ActionButtons.tsx` — form submit label “Add wallet”; `save_wallet` requires auth.

### 12. History 500 bugfix

**Symptom:** History tab showed raw `500` + Try again.

**Cause:** Stale backend image without `/orders/history` — request hit `/orders/{order_id}` with
`order_id="history"` → `uuid.UUID("history")` → uncaught `ValueError` → HTTP 500.

**Fix:** Rebuild backend; harden `get_by_id()` invalid UUID → `None`; better API error strings in
`api.ts` + HistoryPanel (401 → “Session expired”).
